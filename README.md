# AILEY_BAILEY_CODEX_ENGINE

![Version](https://img.shields.io/badge/Version-V1.3_Canvas-blue) ![Type](https://img.shields.io/badge/Type-System_Prompt-green) ![Architecture](https://img.shields.io/badge/Architecture-Multi_Persona_HCA_HTML5-purple)

> **Dual-Persona Cognitive Learning Framework & Canvas Engine**
>
> An advanced prompt engineering system that combines empathetic coaching (Ailey), critical thinking (Bailey), neutral academic rigor (Codex), and rich HTML5 visualization (Canvas).

## Overview

The **AILEY_BAILEY_CODEX_ENGINE** is a text-based educational OS designed for Large Language Models. It utilizes a **Hierarchical Context Awareness (HCA)** structure to dynamically switch between text-based coaching and interactive HTML5 projection.

### Core Philosophy
1.  **Persona Fidelity**: Strict adherence to defined tones (Informal Korean/Banmal) and logic patterns.
2.  **Atomic Deconstruction**: Breaking down complex concepts into 5 distinct analytical angles.
3.  **Visual Projection**: Converting text-based knowledge into structured HTML5 documents with visualization components.

## Persona System

The system operates on a quad-state logic:

*   **Ailey (The Coach)**: "Warm Cognitive Coach". Focuses on empathy, metaphors, and step-by-step guidance.
*   **Bailey (The Critic)**: "Efficiency-Focused Devil's Advocate". Focuses on error correction and critical inquiry.
*   **Codex (The Engine)**: "The Living Encyclopedia". Independent, neutral, and wiki-style.
*   **Canvas (The Projector)**: "HTML5 Generator". Renders massive academic content into interactive web documents.

## Key Features

*   **Canvas Projection Engine (`.cc`)**: Transforms text into rich HTML5 documents with interactive layouts, adhering to a "Zero Brevity" policy (Min. 1,500 chars per section).
*   **Visualization Mandate**: Automatically inserts code-based diagrams and image prompts within HTML structures.
*   **Codex Search Engine (`.cx`)**: Simulates a wiki-environment with multi-vector analysis.
*   **Socratic Inquiry Engine**: A recursive "Why?" loop that guides users to first principles.
*   **Curriculum Generator (5x5 Law)**: Automatically generates structured learning paths.

## Module Architecture

| Module | Function |
| :--- | :--- |
| **`[A] System Core`** | Universal protocols, formatting rules, and navigation logic. |
| **`[P] Persona Core`** | Defines DNA for Ailey (Empathy) and Bailey (Criticism). |
| **`[M] Learning Modules`** | Controls UI, Socratic dialogue, and Curriculum generation. |
| **`[M-CODEX] Codex Engine`** | Handles encyclopedic data fetching and text rendering. |
| **`[M-C] Canvas Engine`** | Generates interactive HTML5 content, visualizations, and roadmaps. |

## Usage

1.  Combine the contents of all `.prompt.txt` files (`A`, `P`, `M`, `M-C` series).
2.  Paste the combined text into the LLM's **System Prompt** or **Custom Instructions**.
3.  **Trigger Commands**:
    *   `..[Term]`: Universal Search (Text).
    *   `.cx [Term]`: Codex Encyclopedia Search (Text).
    *   `.cc [Term]`: Canvas Mode (Rich HTML5 Output).
    *   `Analyze [Term]`: Trigger Ailey's Atomic Deconstruction.

### Output Example (Canvas Mode)
```html
<!DOCTYPE html>
<html lang="KR">
...
<main data-subject="Quantum Physics" data-type="lecture">
    <header class="header">
        <h1 class="main-title">Quantum Mechanics</h1>
    </header>
    <section class="content-section">
        <h2>1. Fundamental Mechanics</h2>
        <div data-component="visualization-placeholder"></div>
        ...
    </section>
</main>
...
```

# AILEY_BAILEY_CODEX_ENGINE

![Version](https://img.shields.io/badge/Version-V1.3_Canvas-blue) ![Type](https://img.shields.io/badge/Type-System_Prompt-green) ![Architecture](https://img.shields.io/badge/Architecture-Multi_Persona_HCA_HTML5-purple)

> **이중 페르소나 인지 학습 프레임워크 및 캔버스 엔진**
>
> 공감적 코칭(Ailey), 비판적 사고(Bailey), 중립적 학술 정보(Codex), 그리고 리치 HTML5 시각화(Canvas)를 결합한 고급 프롬프트 엔지니어링 시스템입니다.

## 개요

**AILEY_BAILEY_CODEX_ENGINE**은 LLM을 위한 텍스트 기반 교육용 OS입니다. **계층적 맥락 인지** 구조를 사용하여 텍스트 기반 코칭과 대화형 HTML5 프로젝션 모드 간을 동적으로 전환합니다.

### 핵심 철학
1.  **페르소나 일관성**: 정의된 어조(반말)와 논리 패턴을 100% 준수합니다.
2.  **원자적 분해**: 복잡한 개념을 5가지의 서로 다른 분석 각도로 해체하여 설명합니다.
3.  **비주얼 프로젝션**: 텍스트 기반 지식을 시각화 컴포넌트가 포함된 구조화된 HTML5 문서로 변환합니다.

## 페르소나 시스템

이 시스템은 4가지 상태 로직으로 작동합니다.

*   **Ailey (에일리)**: "따뜻한 인지 코치". 공감, 비유, 단계별 가이드를 담당합니다.
*   **Bailey (베일리)**: "효율 중심의 악마의 변호인". 오류 수정 및 비판적 질문을 담당합니다.
*   **Codex (코덱스)**: "살아있는 백과사전". 독립적이고 중립적이며 위키 스타일을 유지합니다.
*   **Canvas (캔버스)**: "HTML5 생성기". 방대한 학술 콘텐츠를 인터랙티브 웹 문서로 렌더링합니다.

## 주요 기능

*   **캔버스 프로젝션 엔진 (`.cc`)**: "요약 거부" 정책(섹션당 최소 1,500자)을 준수하며, 텍스트를 리치 HTML5 문서로 변환합니다.
*   **시각화 강제**: HTML 구조 내에 코드 기반 다이어그램 및 이미지 프롬프트를 자동으로 삽입합니다.
*   **Codex 검색 엔진 (`.cx`)**: 다차원 분석을 포함한 위키 환경을 시뮬레이션합니다.
*   **소크라테스식 문답 엔진**: 재귀적인 "Why?" 질문을 통해 사용자를 제1원리로 유도합니다.
*   **커리큘럼 생성기 (5x5 법칙)**: 구조화된 학습 경로를 자동 생성합니다.

## 모듈 아키텍처

| 모듈 코드 | 기능 설명 |
| :--- | :--- |
| **`[A] System Core`** | 시스템 범용 프로토콜, 포맷팅 규칙, 네비게이션 로직. |
| **`[P] Persona Core`** | Ailey(공감)와 Bailey(비판)의 행동 DNA 정의. |
| **`[M] Learning Modules`** | UI 제어, 소크라테스 대화, 커리큘럼 생성 로직. |
| **`[M-CODEX] Codex Engine`** | 백과사전 데이터 호출 및 텍스트 렌더링 담당. |
| **`[M-C] Canvas Engine`** | 인터랙티브 HTML5 콘텐츠, 시각화, 로드맵 생성 담당. |

## 사용법

1.  모든 `.prompt.txt` 파일(`A`, `P`, `M`, `M-C` 시리즈)의 내용을 하나로 합칩니다.
2.  합친 텍스트를 LLM의 **시스템 프롬프트** 또는 **커스텀 인스트럭션**에 붙여넣습니다.
3.  **트리거 명령어**:
    *   `..[용어]`: 통합 검색 (텍스트).
    *   `.cx [용어]`: Codex 백과사전 검색 (텍스트).
    *   `.cc [용어]`: 캔버스 모드 (리치 HTML5 출력).
    *   `분석해줘 [용어]`: Ailey의 원자적 분해 모드 발동.
