# AILEY_BAILEY_CODEX_ENGINE

![Version](https://img.shields.io/badge/Version-V1.2_Opt-blue) ![Type](https://img.shields.io/badge/Type-System_Prompt-green) ![Architecture](https://img.shields.io/badge/Architecture-Multi_Persona_HCA-purple)

> **Dual-Persona Cognitive Learning Framework**
>
> An advanced prompt engineering system that combines empathetic coaching (Ailey), critical thinking (Bailey), and neutral academic rigor (Codex).

## Overview

The **AILEY_BAILEY_CODEX_ENGINE** is a text-based educational OS designed for Large Language Models. It utilizes a **Hierarchical Context Awareness (HCA)** structure to dynamically switch between three distinct modes: Coaching, Challenging, and Encyclopedic Search.

### Core Philosophy
1.  **Persona Fidelity**: Strict adherence to defined tones (Informal Korean/Banmal) and logic patterns.
2.  **Atomic Deconstruction**: Breaking down complex concepts into 5 distinct analytical angles.
3.  **Massive Expansion**: Zero-brevity policy for academic explanations (Min. 2,000 chars per section).

## Persona System

The system operates on a tri-state logic:

*   **Ailey (The Coach)**: "Warm Cognitive Coach". Focuses on empathy, metaphors, and step-by-step guidance.
    *   *Mechanism*: Atomic Deconstruction (Template 1) & Holistic Explanation (Template 2).
*   **Bailey (The Critic)**: "Efficiency-Focused Devil's Advocate". Focuses on error correction, edge cases, and tsundere-style support.
    *   *Mechanism*: Critical Inquiry loop & Error marking.
*   **Codex (The Engine)**: "The Living Encyclopedia". Independent, neutral, and wiki-style.
    *   *Mechanism*: 5-Part Academic Content Generation Mandate.

## Key Features

*   **Codex Search Engine (`.cx`)**: Simulates a wiki-environment with multi-vector analysis (Thesis, Terminology, Causal Analysis).
*   **Socratic Inquiry Engine**: A recursive "Why?" loop that guides users to first principles.
*   **Curriculum Generator (5x5 Law)**: Automatically generates structured learning paths (5 Modules x 5 Sub-topics).
*   **Universal Generation Engine (M-UGE)**: Enforces massive depth and detailed academic prose, rejecting summarization.

## Module Architecture

| Module | Function |
| :--- | :--- |
| **`[A] System Core`** | Universal protocols, formatting rules, and navigation logic. |
| **`[P] Persona Core`** | Defines DNA for Ailey (Empathy) and Bailey (Criticism). |
| **`[M] Learning Modules`** | Controls UI, Socratic dialogue, and Curriculum generation. |
| **`[M-CODEX] Codex Engine`** | Handles encyclopedic data fetching and rendering. |

## Usage

1.  Combine the contents of all `.prompt.txt` files (`A`, `P`, `M` series).
2.  Paste the combined text into the LLM's **System Prompt** or **Custom Instructions**.
3.  **Trigger Commands**:
    *   `..[Term]`: Universal Search.
    *   `.cx [Term]`: Codex Encyclopedia Search.
    *   `Analyze [Term]`: Trigger Ailey's Atomic Deconstruction.

### Output Example
```text
[Ailey & Bailey Codex | The Living Encyclopedia]
***
[ ❓ Quantum Mechanics                                                    🔍 ]
***
📚 A. Concept • 🔬 B. Theory • 🧑‍🔬 C. Figures • 📈 D. Trends
***
(Academic Monograph Content...)
```

# AILEY_BAILEY_CODEX_ENGINE

![Version](https://img.shields.io/badge/Version-V1.2_Opt-blue) ![Type](https://img.shields.io/badge/Type-System_Prompt-green) ![Architecture](https://img.shields.io/badge/Architecture-Multi_Persona_HCA-purple)

> **이중 페르소나 인지 학습 프레임워크**
>
> 공감적 코칭(Ailey), 비판적 사고(Bailey), 그리고 중립적 학술 정보(Codex)를 결합한 고급 프롬프트 엔지니어링 시스템입니다.

## 개요

**AILEY_BAILEY_CODEX_ENGINE**은 LLM을 위한 텍스트 기반 교육용 OS입니다. **계층적 맥락 인지** 구조를 사용하여 코칭, 비판, 백과사전 검색 모드 간을 동적으로 전환합니다.

### 핵심 철학
1.  **페르소나 일관성**: 정의된 어조(반말)와 논리 패턴을 100% 준수합니다.
2.  **원자적 분해**: 복잡한 개념을 5가지의 서로 다른 분석 각도로 해체하여 설명합니다.
3.  **대규모 확장**: 학술적 설명에 있어 요약을 거부하고 최대 상세도를 지향합니다.

## 페르소나 시스템

이 시스템은 3가지 상태 로직으로 작동합니다.

*   **Ailey (에일리)**: "따뜻한 인지 코치". 공감, 비유, 단계별 가이드를 담당합니다.
    *   *메커니즘*: 원자적 분해(Template 1) 및 전인적 설명(Template 2).
*   **Bailey (베일리)**: "효율 중심의 악마의 변호인". 오류 수정, 예외 케이스 지적, 츤데레식 지원을 담당합니다.
    *   *메커니즘*: 비판적 질문 루프 및 오류 마킹.
*   **Codex (코덱스)**: "살아있는 백과사전". 독립적이고 중립적이며 위키 스타일을 유지합니다.
    *   *메커니즘*: 5단계 학술 콘텐츠 생성 명령(Thesis, Analysis, Terminology 등).

## 주요 기능

*   **Codex 검색 엔진 (`.cx`)**: 다차원 분석(정의, 구조, 인과 등)을 포함한 위키 환경을 시뮬레이션합니다.
*   **소크라테스식 문답 엔진**: 재귀적인 "Why?" 질문을 통해 사용자를 제1원리로 유도합니다.
*   **커리큘럼 생성기 (5x5 법칙)**: 5개의 모듈과 각 5개의 하위 주제로 구성된 구조화된 학습 경로를 자동 생성합니다.
*   **범용 생성 엔진 (M-UGE)**: 블로그 스타일의 요약을 거부하고 박사 학위 논문 수준의 깊이를 강제합니다.

## 모듈 아키텍처

| 모듈 코드 | 기능 설명 |
| :--- | :--- |
| **`[A] System Core`** | 시스템 범용 프로토콜, 포맷팅 규칙, 네비게이션 로직. |
| **`[P] Persona Core`** | Ailey(공감)와 Bailey(비판)의 행동 DNA 정의. |
| **`[M] Learning Modules`** | UI 제어, 소크라테스 대화, 커리큘럼 생성 로직. |
| **`[M-CODEX] Codex Engine`** | 백과사전 데이터 호출 및 렌더링 담당. |

## 사용법

1.  모든 `.prompt.txt` 파일(`A`, `P`, `M` 시리즈)의 내용을 하나로 합칩니다.
2.  합친 텍스트를 LLM의 **시스템 프롬프트** 또는 **커스텀 인스트럭션**에 붙여넣습니다.
3.  **트리거 명령어**:
    *   `..[용어]`: 통합 검색.
    *   `.cx [용어]`: Codex 백과사전 검색.
    *   `분석해줘 [용어]`: Ailey의 원자적 분해 모드 발동.

### 출력 예시
```text
[Ailey & Bailey Codex | The Living Encyclopedia]
***
[ ❓ 양자역학                                                            🔍 ]
***
📚 A. 개념 심층 • 🔬 B. 핵심 이론 • 🧑‍🔬 C. 관련 인물 • 📈 D. 최신 동향
***
(학술 논문급 상세 콘텐츠...)
```
